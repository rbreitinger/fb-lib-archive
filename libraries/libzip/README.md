# libzip 1.0.1

ZIP archive read/write library.  
FreeBASIC binding: `zip.bi` (ships with FB 1.10.1)

---

## What is libzip?

libzip is a C library for reading, creating, and modifying ZIP archives.
It supports deflate compression (via zlib), encryption, in-memory sources,
and streaming. It is unrelated to zlib -- it uses zlib internally for
compression but operates at the archive level, not the byte-stream level.

## Dependencies

libzip depends on zlib at link time (`#inclib "z"` in `zip.bi`).
You need zlib binaries alongside libzip. The zlib 1.2.8 bundle in this
archive is the correct companion.

## Usage

### Dynamic linking (recommended for FreeBASIC)

Place `libzip.dll` and `zlib1.dll` next to your executable, or in a
folder on PATH. Compile with:

```
fbc32 yourprogram.bas -p path/to/libzip-1.0.1/win32/lib -p path/to/zlib-1.2.8/win32/lib
```

The `#inclib "zip"` and `#inclib "z"` directives in `zip.bi` handle the
rest automatically.

### Static linking (MinGW only -- NOT supported with stock FreeBASIC)

`libzip.a` is included for users with a full MinGW toolchain. It cannot
be used for static linking with the stock FreeBASIC compiler because
libzip was built with MinGW GCC 14.x which targets the UCRT runtime,
while FB's linker targets the older MSVCRT. Attempting static linking
with `fbc32` produces undefined reference errors such as:

```
undefined reference to `__stdio_common_vswprintf'
```

For FreeBASIC use, always link dynamically via `libzip.dll.a`.

## Build notes (win32)

Built from source using CMake + MinGW32 (GCC 14.2.0, CodeBlocks bundle).
zlib headers sourced from zlib-1.2.8 original source tree.
zlib import library sourced from the zlib-1.2.8 win32 bundle in this archive.

CMake command used:
```
cmake -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release -DBUILD_SHARED_LIBS=ON
      -DZLIB_INCLUDE_DIR=/path/to/zlib-1.2.8
      -DZLIB_LIBRARY=/path/to/zlib-1.2.8/win32/lib/libz.dll.a
      ..
mingw32-make
```
